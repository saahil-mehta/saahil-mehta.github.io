SCD2, but only where it means something

History is kept for analytically meaningful changes: segment, KYC, suitability, vulnerability, residency. Corrections that should not create history stay type 1.

Type 4 mini-dimensions for the volatile

Monthly-refreshed scores would explode a multi-million-row party dimension. Banded scores move to small mini-dimensions (dim_party_risk_behaviour) keyed on the fact.

Discrete scores stay as facts

Banding loses precision, so the raw score is also stored as a fact for data science, and a type 6 current-profile column serves "as-is-now" questions cheaply.

09
Metrics as code, not as seven dashboards

  
One audited definition per metric, versioned in git, compiled to SQL at query time. Flip a card.

  

  
A metric is not certified until it carries: owner · formula · grain · aggregation behaviour · filters and exclusions · currency basis · time basis · lineage · test coverage · version · effective date · approved consumers. Each is served with a "data as-of" watermark so the SLA is visible at the point of use.

Fast for the dashboard, frugal for the bank

  
Partition pruning does the heavy lifting. Run a query against 18 months of events and watch the cost fall.

Within each partition, files are clustered by account_sk, party_sk, product_sk, channel, transaction_status, so the most common predicates prune again inside the partition. Executive dashboards never touch this table at all: they read aggregate marts built once per cycle.

  

  
    
    
  • Tag compute jobs, models, schemas and dashboards by domain, owner, environment and cost centre, so cost per data product is reportable.
    • 
    
  • Promote repeated expensive query logic into certified intermediate or aggregate models; materialise the right grain once, reuse it everywhere.
    • 
    
  • Separate dev, test and prod compute; slim CI builds only changed models against row-limited fixtures.
    • 
    
  • Budgets and anomaly alerts catch runaway queries, accidental full scans and sandbox growth before the invoice does.
    • 
    
  • Iceberg maintenance runs as scheduled jobs: compaction, orphan-file removal, metadata cleanup, snapshot expiry per retention rules.
